# Workstreams
These are stable lanes for planning and staffing. Releases should be vertical slices across multiple workstreams.

---

## A. Product Surface and Workflow
- Projects (class/unit), artifact lifecycle (draft → reviewed → approved)
- Editor UX, citation inspector, QA report viewer, fix/regenerate flows

## B. Book Knowledge Base (KB)
- Ingestion pipeline (PDF/EPUB/DOCX), structure extraction, chunking
- Lexical + vector indexing
- License/excerpt policy enforcement

## C. Generation and Orchestration
- Retrieval planner (hybrid)
- Agent orchestrator (DGA → SVA → QAA), gating, remediation loops
- Structured schemas for artifacts and reports

## D. Teacher Style System
- Teacher doc ingestion, PII redaction, style profile extraction/versioning
- Exemplar indexes, style-conditioned generation, anti-copy enforcement

## E. Item Bank and Reuse
- Save/tag/approve items
- Search/filter, near-duplicate detection
- Usage analytics

## F. Export and Packaging
- PDF and DOCX export templates
- Watermarking, formatting rules
- (Later) LMS export formats

## G. Platform, Security, Observability
- RBAC, tenant isolation, encryption, audit logs
- OpenRouter integration, routing policies, fallback, telemetry
- Cost controls, rate limiting, caching, monitoring
